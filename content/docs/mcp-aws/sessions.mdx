title: Sessions
description: Per-session state with Memory and DynamoDB stores
---

MCP sessions track tool visibility, gate state, call history, and auth context. Each session is identified by the `x-session-id` header.

## Memory Store (Dev/Test)

```ts
import { MemorySessionStore } from "@stratus/mcp-aws";

server.lambda({
  sessionStore: new MemorySessionStore({
    maxSessions: 10_000,   // evict oldest when exceeded
    ttlMs: 3_600_000,      // 1 hour TTL
  }),
});
```

Default when no store is specified. Data lost on Lambda cold start.

## DynamoDB Store (Production)

```ts
import { DynamoSessionStore } from "@stratus/mcp-aws/dynamo";

server.lambda({
  sessionStore: new DynamoSessionStore({
    tableName: "mcp-sessions",
    region: "us-east-1",
    ttlSeconds: 86400,  // 24 hours
  }),
});
```

Table schema: partition key `pk` (String). The store handles serialization of Sets, arrays, and nested objects.

### Create the Table

```bash
aws dynamodb create-table \
  --table-name mcp-sessions \
  --attribute-definitions AttributeName=pk,AttributeType=S \
  --key-schema AttributeName=pk,KeyType=HASH \
  --billing-mode PAY_PER_REQUEST
```

Enable TTL on the `expiresAt` attribute for automatic cleanup:

```bash
aws dynamodb update-time-to-live \
  --table-name mcp-sessions \
  --time-to-live-specification Enabled=true,AttributeName=expiresAt
```

## SQLite (ECS/Fargate/EC2)

Uses Bun's native `bun:sqlite`. Zero dependencies, file-based persistence.

```ts
import { SqliteSessionStore } from "@stratus/mcp-aws";

server.lambda({
  sessionStore: new SqliteSessionStore({
    path: "/tmp/mcp-sessions.db",  // default: ":memory:"
    ttlMs: 3_600_000,              // 1 hour
  }),
});
```

Ideal for container deployments where DynamoDB is overkill. Supports file persistence across restarts when a path is provided.

## Session State

Each session tracks:

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Session identifier |
| `visibleTools` | `Set<string>` | Tools promoted by search or gate unlock |
| `unlockedGates` | `Set<string>` | Gates that have been satisfied |
| `toolCallHistory` | `ToolCallRecord[]` | Audit trail of tool calls |
| `auth` | `AuthContext` | User identity and roles |
| `metadata` | `Record<string, unknown>` | Developer-defined data |

## Custom Store

Implement the `SessionStore` interface:

```ts
import type { SessionStore, McpSession } from "@stratus/mcp-aws";

class RedisSessionStore implements SessionStore {
  async get(sessionId: string): Promise<McpSession | undefined> {
    const data = await redis.get(`mcp:${sessionId}`);
    return data ? deserialize(data) : undefined;
  }

  async set(session: McpSession): Promise<void> {
    await redis.set(`mcp:${session.id}`, serialize(session), "EX", 3600);
  }

  async delete(sessionId: string): Promise<void> {
    await redis.del(`mcp:${sessionId}`);
  }
}
```

## Session Continuity

Pass a consistent `x-session-id` header to maintain state across Lambda invocations:

```ts
// Request 1: search promotes tools
fetch(url, { headers: { "x-session-id": "abc" }, body: searchRequest });

// Request 2: promoted tools are available
fetch(url, { headers: { "x-session-id": "abc" }, body: toolCallRequest });
```

Without the header, each request gets a unique session (no state carries over).
