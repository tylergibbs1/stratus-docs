title: Authentication
description: API keys, Cognito JWT, and AsyncLocalStorage context

Auth is configured once on the server. All transports inherit it.

## API Key

```ts
import { apiKey } from "@usestratus/mcp-aws";

server.auth(apiKey({
  "sk-live-abc123": { subject: "user-1", roles: ["admin"], claims: { org: "acme" } },
  "sk-live-xyz789": { subject: "user-2", roles: ["reader"] },
}));
```

Checks the `x-api-key` header by default. Custom header:

```ts
server.auth(apiKey({ "my-token": { roles: ["admin"] } }, { headerName: "authorization" }));
```

## Cognito JWT

```ts
import { cognito } from "@usestratus/mcp-aws";

server.auth(cognito({
  userPoolId: "us-east-1_abc123",
  region: "us-east-1",
  audience: "my-client-id",      // optional
  rolesClaim: "cognito:groups",  // optional, default
}));
```

Uses `jose` with `createRemoteJWKSet` — JWKS is cached and rotated automatically.

## Chaining Providers

Multiple auth providers try in order, first success wins:

```ts
server.auth(
  apiKey({ "sk-123": { roles: ["admin"] } }),
  cognito({ userPoolId: "...", region: "us-east-1" }),
);
```

Or chain with multiple `.auth()` calls:

```ts
server
  .auth(apiKey({ "sk-123": { roles: ["admin"] } }))
  .auth(cognito({ userPoolId: "...", region: "us-east-1" }));
```

## AsyncLocalStorage Context

Tool handlers can access auth without explicit parameters:

```ts
import { getAuthContext, getSession } from "@usestratus/mcp-aws";

server.tool("my_tool", async () => {
  const auth = getAuthContext();
  if (!auth.authenticated) return "Please authenticate";

  const session = getSession();
  return `Hello ${auth.subject}, session ${session?.id}`;
});
```

Works anywhere in the call stack — nested function calls, utility modules, etc. No need to thread `ctx` through every function.

## AuthContext Shape

```ts
type AuthContext = {
  authenticated: boolean;
  subject?: string;       // user ID
  roles: string[];        // for role() gates
  claims: Record<string, unknown>; // full token claims
};
```

## Custom Provider

```ts
import type { AuthProvider } from "@usestratus/mcp-aws";

const myAuth: AuthProvider = {
  async authenticate(request) {
    const token = request.headers["x-custom-token"];
    if (!token) return { authenticated: false, roles: [], claims: {} };

    const user = await verifyToken(token);
    return {
      authenticated: true,
      subject: user.id,
      roles: user.roles,
      claims: user,
    };
  },
};

server.auth(myAuth);
```

## 401 Response

When auth fails, the Lambda handler returns:

```json
{
  "statusCode": 401,
  "headers": {
    "WWW-Authenticate": "Bearer realm=\"mcp-server\", resource_metadata=\"https://api.example.com/.well-known/oauth-protected-resource\""
  },
  "body": "{\"jsonrpc\":\"2.0\",\"error\":{\"code\":-32000,\"message\":\"Authentication required\"},\"id\":null}"
}
```

The `resource_metadata` URL points to the [RFC 9728 OAuth metadata](/mcp-aws/security#rfc-9728-oauth-metadata) endpoint.
