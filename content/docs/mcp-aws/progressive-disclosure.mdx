---
title: Progressive Disclosure
description: Only show agents the tools they need
---

MCP servers with 50+ tools flood the agent's context window. Progressive disclosure shows only relevant tools, with the rest discoverable via search.

## Tier System

| Tier | In `tools/list`? | Searchable? | When to use |
|------|-------------------|-------------|-------------|
| `always` | Yes | N/A | Core tools (3-5 max) |
| `discoverable` | No | Yes | Most tools |
| `hidden` | No | No | Gated/prerequisite tools |

```ts
server
  .tool("navigate", handler)                                             // always (default)
  .tool("click", { tier: "discoverable", tags: ["interaction"] }, handler) // searchable
  .tool("execute_trade", { tier: "hidden", gate: requires("review") }, handler) // gated
```

## How It Works

1. Client calls `tools/list` → gets only `always`-tier tools + `search_tools`
2. Agent calls `search_tools("click form input")` → BM25 search matches tools
3. Matched tools promoted to session → server sends `tools/list_changed`
4. Client re-fetches `tools/list` → sees promoted tools alongside always-tier
5. Visibility is **per-session** — other sessions are unaffected

Works with Claude Desktop, Cursor, Windsurf, or any client that supports `tools/list_changed`.

## Search Index

Built automatically from tool names, descriptions, tags, and parameter names. BM25 ranking.

```ts
// Tags improve search relevance
server.tool("fill_form", {
  description: "Fill a form field with a value",
  tier: "discoverable",
  tags: ["form", "input", "interaction", "fill"],
}, handler);
```

## Disclosure Modes

Auto-inferred from your tool tiers. No config needed.

| Your tools | Inferred mode | `tools/list` shows |
|---|---|---|
| All `always` | `all` | Every tool |
| Any `discoverable` or `hidden` | `progressive` | `always` tools + `search_tools` |
| Explicit `code-first` config | `code-first` | `search_tools` + `execute_workflow` only |

Override explicitly:

```ts
const server = new McpServer({
  name: "tools",
  version: "1.0.0",
  disclosure: { mode: "code-first" },
  codeMode: { enabled: true },
});
```

## Session-Scoped Visibility

Promoted tools are tracked per-session. If you use a `DynamoSessionStore`, promotions persist across Lambda invocations for the same session.

Pass a consistent session ID via the `x-session-id` header:

```ts
// First request: search
fetch(url, { headers: { "x-session-id": "session-123" }, body: searchRequest });

// Second request: use promoted tool
fetch(url, { headers: { "x-session-id": "session-123" }, body: toolCallRequest });
```
